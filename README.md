# Inventory Chatbot

#### Starting from the "State", it contains the following:
- message: message history between the bot and the user
- question: the question asked by the user or the input, can be either chitchat or a question about SQL query
- sql_query: the SQL query generated by the LLM
- sql_result: the result retrieved by the SQL query
- error: if the LLM fails to generate a SQL query
- intent: the intent of the user's message

#### Then for the nodes, there are six:

1- intent_node: given the INTENT_PROMPT and the human message, it will determine the intent as either "CHITCHAT" or "generator"

2- chit_chat_node: given the CHITCHAT_PROMPT and the human message, it will generate a chit-chat response without referring to the database

3- sql_generator_node: given the SYSTEM_PROMPT and the human message, it will generate a SQL query without referring to the database

4- sql_executor_node: it fetches the sql_query from the state and executes it

5- sql_corrector_node: it corrects and refines the SQL query if an error occurs in the state, until the query works

6- responder_node: given the RESPONSE_PROMPT and the SQL result, it will generate a response to the user

After that, the graph was constructed. Two additional functions were created to help control the flow of the program:
- executor_should continue: the function used in the conditional edge of the sql_executor_node, if it returns "error", 
it should keep on calling the responder until it reaches a correct response and returns "responder"
- intent_should_continue: the function used in the routing of the intent_node, if the intent is "generator," it goes to 
the sql_generator_node, if it's "chitchat" goes to the chit_chat_node. It repeats this with every message.
